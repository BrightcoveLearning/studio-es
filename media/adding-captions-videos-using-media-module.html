Las propiedades de las pistas de texto se pueden configurar para proporcionar subtítulos para un video. Los subtítulos brindan información adicional o interpretativa a las personas que desean acceder a ellos. Los subtítulos son necesarios para que las personas sordas y con problemas de audición puedan acceder a la parte de audio de las presentaciones de video. Para mostrar los subtítulos de un video en el reproductor Brightcove, debe completar las siguientes tareas:

  • Crear archivos de subtítulos
  • Configurar las propiedades de la pista de texto
  • Leyendas de estilo

Crear archivos de subtítulos

Para agregar subtítulos a un video, primero debe crear un archivo de subtítulos. El reproductor Brightcove es compatible con los formatos Web Video Text Tracks (WebVTT), SRT, DFXP y SCC. El formato de archivo WebVTT (que se muestra a continuación) es texto sin formato y contiene datos de texto de tiempo seguidos del título o subtítulo deseado.

Notas

  • Los subtítulos SRT, SCC y DFXP se convertirán automáticamente a WebVTT para las cuentas de Dynamic Delivery.
  • El texto de los subtítulos debe estar codificado en UTF-8 para que se muestre correctamente en idiomas de doble byte como japonés y coreano.
  • Debido a que los subtítulos se muestran en HTML, cualquier texto dentro < > los corchetes se interpretarán como una etiqueta y no se mostrarán.
WEBVTT 00: 00: 00.000 -> 00: 00: 07.080 alinear: línea media: 90% 00: 00: 07.080 -> 00: 00: 10.280 align: middle line: 84% Hola, soy Bob Bailey, un especialista en aprendizaje de Brightcove. 00: 00: 10.280 -> 00: 00: 12.780 align: middle line: 84% En este video, aprenderá cómo Video Cloud Studio es 00: 00: 12.780 -> 00: 00: 14.940 align: middle line: 84% organizado y obtenga una breve descripción de lo que 00: 00: 14.940 -> 00: 00: 17.490 align: middle line: 90% que puedes lograr usándolo.
Notas:
  • Los nombres de los archivos de subtítulos solo deben incluir letras (mayúsculas o minúsculas), números, guiones (-) y guiones bajos (_).
  • Hemos visto casos en los que los subtítulos creados por un tercero comienzan los subtítulos con la marca de tiempo:

    01: 00: 00.000

    Esto hará que los subtítulos sean una hora en el video. Para asegurarse de que los subtítulos estén sincronizados correctamente con el video, asegúrese de que las marcas de tiempo comiencen con 00: 00: 00.000 y compruebe todas las marcas de tiempo para 01 compensación horaria.

Para obtener más información sobre el formato WebVTT, consulte este enlace. Debe tener un archivo de subtítulos separado para cada idioma que necesite admitir. Si bien existen muchas herramientas de terceros para ayudar a crear archivos de subtítulos, Brightcove tiene varios socios que brindan servicios de transcripción, subtítulos y traducción. Para obtener una lista completa de los socios de Brightcove, visite el Socios sección de nuestro sitio web.

Configurar las propiedades de la pista de texto

Una vez que se han creado los archivos de subtítulos, Brightcove puede cargarlos y alojarlos o residir en una URL accesible en Internet.

Para ver y editar las propiedades de la pista de texto de un video, abra el módulo de Medios, haga clic en el enlace del título del video y ubique el Pistas de texto sección. Para configurar las propiedades de la pista de texto, siga estos pasos:

  1. Hacer clic Editar o haga doble clic en la sección.
  2. Seleccione si el archivo de la pista de texto se cargará en Video Cloud o se alojará en una URL externa.
    • Subir archivo - Haga clic en Navegar y seleccione el archivo de pista de texto del sistema de archivos. El archivo se cargará en Video Cloud.
    • Subir URL - Ingrese la URL pública donde reside el archivo de pista de texto y luego haga clic en Subir. El archivo se cargará en Video Cloud.
    • URL remota - Ingrese la URL pública donde reside el archivo de pista de texto. La URL debe tener menos de 250 caracteres. Los subtítulos se cargarán desde la URL remota.
      Nota: Si aloja archivos de subtítulos en su propio servidor, asegúrese de que el archivo esté ubicado en una URL a la que pueda acceder el reproductor de Brightcove. El servidor que aloja el archivo de subtítulos también debe tener habilitado el uso compartido de recursos entre orígenes (CORS). Puedes leer más sobre CORS aquí.
      Nota: Los nombres de archivo de pistas de texto con paréntesis, signos de interrogación, comillas, espacios y otros caracteres especiales fallarán silenciosamente al cargar. Brightcove recomienda utilizar nombres de archivo sin caracteres especiales.
      Nota: La pista de texto de los vídeos compartidos en la cuenta no se puede editar en el módulo multimedia.
    Nota: los Predeterminado (visualización automática) La opción actualmente no tiene ningún efecto en los títulos de Dynamic Delivery.
  3. La Idioma El campo debe contener el código de idioma de dos letras que especifica el idioma de los datos de texto de la pista.
  4. Ingrese el apropiado Etiqueta para el archivo de subtítulos. La etiqueta se muestra dentro del reproductor y permite al espectador seleccionar un idioma. Si no se proporciona ninguna etiqueta, se muestra el código de idioma.
  5. Selecciona el Amable de pista de texto. Las opciones incluyen:
    • Subtítulos - Similar a los subtítulos, pero también puede incluir efectos de sonido y otra información de audio
    • Subtítulos - La transcripción o traducción del diálogo
    • Descripciones - Destinado a ser un archivo de texto separado que describe el video a través de un lector de pantalla
    • Capítulos - Destinado a ayudar al usuario a navegar por el video
    • Metadatos - Información y contenido sobre el video que no está destinado a mostrarse al espectador de forma predeterminada (aunque es posible que desee hacerlo mediante JavaScript)
  6. Introducir el Tipo de Mimica para el archivo de pista de texto. Al seleccionar el valor predeterminado, se mostrarán los subtítulos cuando se reproduce el vídeo (Brightcove Players 6.x y versiones posteriores).
    Tenga en cuenta que si está cargando subtítulos SRT o DFXP, no podrá editar el tipo MIME.
  7. Cheque Predeterminado (visualización automática) para que la pista sea la predeterminada. La mayoría de los navegadores intentarán mostrar primero la pista de texto predeterminada, pero la configuración de idioma y accesibilidad puede anular la configuración predeterminada.
    Nota: La configuración predeterminada no funciona con IE11.
  8. Hacer clic Agregar pista de texto para guardar la información de la pista de texto.
  9. Hacer clic Agregar pista de texto para agregar otra pista de texto o Ahorrar para guardar las pistas de texto.
    Nota: Los cambios realizados en las propiedades de la pista de texto pueden tardar hasta 15 minutos en aparecer en los reproductores publicados.

Si alguna pista de texto se ha designado como predeterminada, aparecerá en negrita. Para eliminar una pista de texto, haga clic en el icono de eliminar () junto a la pista de texto.

Tenga en cuenta que el default el valor no es por pista, pero se establece en true para todas las pistas del mismo idioma (srclang)

Cuando se publica un reproductor con subtítulos asociados, un CC Aparecerá el botón en el cromo del reproductor. Haga clic en el CC para seleccionar un idioma de subtítulos y mostrar los subtítulos.

Leyendas de estilo

La apariencia predeterminada de los subtítulos está sujeta a cambios, por lo que es posible que desee modificar el estilo de los subtítulos. Para obtener información sobre el estilo de los subtítulos, consulte Estilo de subtítulos.

Limitaciones

  • Los subtítulos son compatibles con los siguientes tipos de dispositivos y sistemas operativos.
    • iOS 10+
    • Android 7+
    • Internet Explorer 11 o superior
    • Los últimos navegadores Chrome, Safari y Firefox en computadoras de escritorio
  • No hay límite técnico para la longitud de un título, pero para que el título completo sea visible, el reproductor debe ser lo suficientemente grande para mostrar esa cantidad de texto.
  • Debido a que Safari usa las capacidades nativas para diseñar subtítulos, el Configuración de subtítulos La opción de menú no está disponible en el CC en los navegadores Safari (iOS y macOS).
  • Si ingiere un archivo WebVTT no válido en Studio, el archivo se "procesará", pero no se agregará al video. Solución alternativa: puede validar sus archivos WebVTT utilizando este validador en línea.
