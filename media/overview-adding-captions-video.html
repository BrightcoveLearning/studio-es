Introducción

Se pueden agregar subtítulos a los videos para proporcionar información adicional o interpretativa a las personas que deseen acceder a ellos. Los subtítulos son necesarios para que las personas sordas y con problemas de audición puedan acceder a la parte de audio de las presentaciones de video.

Dicho texto puede usarse para accesibilidad, para proporcionar traducciones de la pista de audio o para algún otro propósito, pero la tecnología subyacente utilizada es la misma.

Ubicación del archivo de subtítulos

Puede elegir dónde alojar el archivo de subtítulos WebVTT:

  1. En una URL remota (activo remoto) - En este caso, el archivo WebVTT está alojado en su servidor y agrega la URL del archivo a los metadatos del video.
    Nota: Si aloja archivos de subtítulos en su propio servidor, asegúrese de que el archivo esté ubicado en una URL a la que pueda acceder el reproductor de Brightcove. El servidor que aloja el archivo de subtítulos también debe tener habilitado el uso compartido de recursos entre orígenes (CORS). Puedes leer más sobre CORS aquí.
  2. Cargado mediante Video Cloud (ingerido) : en este caso, el archivo WebVTT se carga (ingiere) en Video Cloud. Esto se puede hacer usando el módulo de Medios o usando las API.
Nota: Si el archivo de origen de video que ingiere en Video Cloud tiene subtítulos 608/708 incrustados, se ignorarán durante el procesamiento del video y no presente en las interpretaciones de video. Para tener subtítulos con tus vídeos, tendrás que agregarlos a través del módulo Media, la API de ingesta dinámica o la API de CMS (para subtítulos alojados de forma remota)

Asociación de subtítulos con un video

Hay varias formas de asociar un archivo de subtítulos con un video:

  1. Usando Studio
  2. Modificar el código de publicación del reproductor
  3. Usando las API

Agregar subtítulos en Studio

Los subtítulos se pueden asociar con un video usando el módulo de Medios. Para obtener información, consulte Agregar subtítulos a videos usando el módulo de medios.

Modificar el código de publicación del reproductor

En este caso, un <track> La etiqueta se agrega al código de publicación del reproductor. Para más información, ver Agregar subtítulos a videos mediante programación.

Agregar subtítulos a través de las API

La forma de agregar subtítulos a través de las API varía según si el archivo de subtítulos se ingiere o se usa como un activo remoto.

  • Subtítulos de activos remotos - Si prefiere alojar los subtítulos usted mismo (o tener un proveedor de subtítulos que los aloje), puede asociar los subtítulos con un video utilizando la API de CMS. Para obtener información, consulte Usando la API de CMS: Agregar subtítulos de WebVTT.

  • Agregar subtítulos a Video Cloud - Si desea que Brightcove aloje sus subtítulos, puede ingerir su archivo WebVTT utilizando la API de ingesta dinámica. Para obtener información, consulte Ingesta de archivos WebVTT.

Nota: Independientemente del método utilizado para añadir archivos de subtítulos a un vídeo, no se pueden añadir más de 1000 pistas de texto.

Eliminar subtítulos

Los subtítulos se pueden eliminar de los videos mediante el módulo de medios o mediante la API de CMS. En Studio, abra el módulo de Medios y edite las propiedades del video. Edite el Pistas de texto propiedad según sea necesario.

Los subtítulos también se pueden eliminar con el API de CMS haciendo un Actualizar video solicitud y enviando un cuerpo de solicitud con un vacío text_tracks matriz (o una matriz de la que se ha eliminado el objeto asociado con ese conjunto de títulos).

Leyendas de estilo

El estilo se puede aplicar a los subtítulos WebVTT usando CSS. Ver Estilo de subtítulos para detalles.
