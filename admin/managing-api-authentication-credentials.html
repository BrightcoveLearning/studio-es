Cuando utilice las API de Video Cloud, sus llamadas de método requerirán tokens de acceso a menos que habilite la autenticación básica en la Configuración del servicio del reproductor. Para obtener más información sobre cómo habilitar la autenticación básica, consulte Administrar la configuración de la cuenta.

Nota: Esta función solo está disponible para cuentas Professional y Enterprise.

Creando registros de clientes

La página de autenticación de API se utiliza para crear nuevos registros de clientes. Al crear registros, puede seleccionar el servicio y los roles que se habilitarán con el registro.

Para crear un nuevo registro de cliente para su cuenta:

  1. Haga clic en el ADMINISTRACIÓN enlace en el encabezado de Studio.
  2. Haga clic en el Autenticación API Enlace. Se abrirá la página de autenticación de API mostrando los registros de sus clientes actuales.
    Nota: Solo se mostrarán los registros de clientes de la cuenta en la que ha iniciado sesión.
  3. Hacer clic Registrar nueva aplicación.
  4. Entrar a Nombre y Descripción para el registro del cliente.
  5. Seleccione una o más de sus cuentas para la autorización.
  6. Seleccione las API y los permisos para habilitar este registro.
    Notas:
  7. Para la integración de DFP / Google Ad Manager, los permisos de API necesarios son Lectura de video CMS.
  8. La selección de una gran cantidad de cuentas y API puede hacer que el tamaño del encabezado HTTP sea grande y no sea válido. El tamaño máximo de encabezado que acepta nuestro servidor es 8K, lo que significa que debe seleccionar 30 cuentas o menos para aplicar las credenciales. Por motivos de seguridad, le recomendamos que limite el alcance de los permisos para un conjunto de credenciales al conjunto más pequeño de cuentas y operaciones necesarias.
  9. Hacer clic Ahorrar.
  10. A Identificación del cliente y Secreto del cliente Será devuelto.
    Nota: Copie el secreto del cliente y guárdelo en una ubicación segura, ya que el secreto del cliente no se volverá a mostrar.

Gestión de registros de clientes

La página de autenticación de API se puede utilizar para ver y administrar los registros de clientes. Haga clic en el ADMINISTRACIÓN enlace en el encabezado de Studio y luego haga clic en Autenticación API. Aparecerá una lista de sus registros actuales.

Para ver los detalles del registro, haga clic en el nombre. Desde aquí puedes:

  • Modifica el nombre
  • Copie la identificación del cliente
  • Agregar cuentas para autorización
  • Seleccione API adicionales para habilitar
  • Eliminar la credencial

Hacer clic Ahorrar para guardar sus cambios.

Tenga en cuenta que si se crea una credencial para varias cuentas, los usuarios solo verán esa credencial si tienen acceso a todas esas cuentas. Por ejemplo, el usuario 1 crea una credencial para las cuentas A y B. El usuario 2 inicia sesión y solo tiene acceso a la cuenta A. El usuario 2 no verá la credencial porque no tiene acceso a la cuenta B.

Eliminar registros de clientes

Para eliminar el registro de un cliente, haga clic en el icono de eliminar () junto al registro o edite el registro y haga clic en Eliminar registro. Eliminar un registro lo deshabilita permanentemente y no se puede deshacer.

Trabajar con DFP

Si actualmente tiene varias cuentas de Video Cloud configuradas como fuentes de contenido en DFP, tiene dos opciones:

  1. Puede crear un ID de cliente y un secreto de cliente únicos para cada cuenta de Video Cloud configurada como fuente de contenido en DFP. Por ejemplo, si tiene 10 cuentas de Video Cloud, tendrá 10 aplicaciones llamadas DFP en la configuración de autenticación de API en Video Cloud.
  2. Puede crear un único ID de cliente y un secreto de cliente para todas las cuentas de Video Cloud que se hayan configurado como fuentes de contenido en DFP. Por ejemplo, si tiene 10 cuentas de Video Cloud, tendrá 1 aplicación llamada DFP en la configuración de autenticación de API en Video Cloud.

Independientemente de si eligió la opción 1 o 2, para cada fuente de contenido que configure en DFP, debe proporcionar un ID de cuenta único. Por lo tanto, no está reduciendo el número de fuentes de contenido que están configuradas actualmente en DFP; simplemente está reemplazando el token de lectura con el ID de cliente y el secreto del cliente, que pueden ser iguales en todas las fuentes de contenido o únicos para cada uno.

La opción n. ° 1 requiere más tiempo desde el principio. Sin embargo, si el ID de cliente y el secreto del cliente son los mismos en todas las fuentes de contenido (como con la opción n. ° 2), será una carga agregar una nueva fuente de contenido porque necesitará generar un nuevo ID de cliente y un secreto de cliente y actualizar todas las fuentes de contenido en DFP. Por lo tanto, Brightcove recomienda la opción n. ° 1.
