Introducción

La API social le permite obtener el estado y el historial de compartir videos en plataformas de redes sociales. Este inicio rápido le permite ver las solicitudes disponibles y sus respuestas, utilizando su propia cuenta o una cuenta de muestra de Brightcove Learning Services.

Notas sobre la aplicación de muestra

  • Este es un comienzo rápido, destinado a brindarle un modelo para crear una aplicación en torno a la API social. Le permite probar solicitudes básicas, pero no emplea ninguno de los parámetros que estas solicitudes pueden usar. Para ver estos, mire el Referencia de API.
  • La aplicación también realiza algunas solicitudes al API de CMS para obtener los nombres de los videos devueltos por la solicitud inicial para el estado de compartir videos en redes sociales. Esto es solo para completar los selectores de video para las otras solicitudes, porque la API social solo devuelve identificadores de video, no sus nombres.
Esta muestra y el código asociado se proporcionan como una guía para su desarrollo de producción. El código no está certificado para uso en producción ni se admite oficialmente.

Obtener credenciales

Para utilizar la API social, necesitará las credenciales adecuadas.

Código de proxy

Para crear su propia versión de la aplicación de muestra en esta página, debe crear y alojar su propio proxy. (Los proxies utilizados por Brightcove Learning Services solo aceptan solicitudes de dominios de Brightcove). Un proxy de muestra, muy similar al que usamos, pero sin las comprobaciones que bloquean las solicitudes de dominios que no son de Brightcove, se puede encontrar en este repositorio de GitHub. También encontrará instrucciones básicas para usarlo allí, y una guía más elaborada para crear aplicaciones alrededor del proxy en Usando las API REST.
