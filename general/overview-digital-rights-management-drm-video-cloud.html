Introducción

La administración de derechos digitales (DRM) evita que sus videos se reproduzcan, excepto en los clientes que tienen permiso para hacerlo. Las implementaciones de DRM varían, pero el mecanismo habitual que se utiliza es encriptar el video y desencriptarlo solo si el cliente tiene algún tipo de clave para autorizar la reproducción.

Para obtener mecanismos de seguridad de contenido adicionales que se pueden configurar en la configuración del reproductor o en las propiedades del video, consulte:

  • Configuración de restricciones de contenido del reproductor
  • Configuración de la disponibilidad de video
  • Protección de videos con cifrado HLS
Notas:
  • Para utilizar el cifrado DRM o HLS, su cuenta debe estar habilitada para estas funciones. Comuníquese con su gerente de cuenta para obtener más información.
  • Cuando los videos están habilitados para DRM, las fuentes / representaciones de MP4 se ocultan automáticamente (lo que significa que existen, pero no se envían al reproductor como fuentes) en Studio, la API de CMS y la API de reproducción.
  • Brightcove actualmente no admite la reproducción DRM de solo video o solo audio.

Tipos de DRM compatibles

Video Cloud ofrece un amplio soporte para DRM:

  • Soporte integral para empaquetado, servicio de licencias y reproducción
  • Uso adaptable del DRM adecuado para un dispositivo determinado:
    • Video Cloud favorece la DRM de dispositivos nativos para minimizar la dependencia de complementos
    • Video Cloud aprovecha los estándares para reducir la cantidad de representaciones necesarias para ofrecer una solución multiplataforma

Brightcove admite los tres tipos más populares de DRM para video HTML5:

  • Juego de Apple FairPlay
  • Google Widevine Modular
  • Microsoft PlayReady

Puede registrarse en varios tipos de DRM para proteger su contenido en todos los dispositivos y plataformas.

    Navegadores

    DRM para navegadores es para escritorio o para web móvil (Chrome en dispositivos Android 6+; Safari en dispositivos iOS 11.2+). Brightcove utiliza MPEG-DASH y cifrado común junto con Widevine Modular, Apple Fairplay y Microsoft PlayReady para proporcionar DRM en el escritorio; el tipo que se utilice depende del navegador. Esto proporciona un único conjunto de activos que se pueden reproducir en diferentes navegadores mientras se aprovecha el DRM nativo que está disponible con los navegadores modernos.

    Aplicaciones iOS y Android

    La siguiente información se aplica a las aplicaciones creadas con el SDK de reproductores nativos de Brightcove. Tenga en cuenta que, además de la reproducción normal, los SDK también admiten reproducción sin conexión con DRM.

    iOS

    Para dispositivos iOS, la solución DRM de Video Cloud actual es HLS con FairPlay Streaming.

    La tecnología FairPlay Streaming (FPS) de Apple proporciona una solución segura para entregar video HTTP Live Streaming (HLS) a dispositivos Apple, incluidos iPhone, iPad y Apple TV. Utiliza un protocolo de clave segura establecido y minimiza el consumo de energía en los dispositivos móviles.

    Androide

    Para los dispositivos Android, la solución es MPEG-DASH + CENC (Widevine Modular).

    Google Widevine ofrece una solución de optimización de video y DRM multiplataforma y multiformato que los propietarios de contenido, proveedores de video por Internet, minoristas y operadores de video pueden utilizar para ofrecer una oferta de entretenimiento de marca y de alta calidad a las plataformas de entretenimiento conectadas a Internet.

    Niveles de seguridad DRM con dispositivos Android

    Widevine implementa un conjunto de estándares de la industria, incluido el cifrado CENC, el intercambio de claves de licencia y la calidad de transmisión adaptativa para administrar la protección de video. Widevine admite varios niveles de calidad de transmisión en función de las capacidades de seguridad del dispositivo receptor.

    Los niveles de seguridad se denominan L1, L2 y L3:

    • Nivel 1 - Nivel de seguridad más alto
      • Las claves claras solo son visibles para el procesador de seguridad
      • Ruta de video protegida. El video claro no está expuesto.
    • Nivel 2 - Menos seguro
      • Las claves claras solo son visibles para el procesador de seguridad
      • Sin ruta de video protegida. Se expone un video claro.
    • Nivel 3 - Nivel de seguridad más bajo
      • Procesamiento de claves realizado por la CPU principal
      • Sin ruta de video protegida. Se expone un video claro.

    Los dispositivos Android admiten niveles de seguridad L1 o L3, según las configuraciones de hardware y software. Solo los dispositivos que cumplen con la especificación L1 pueden transmitir HD o contenido de mayor calidad desde servidores protegidos por Widevine. Si su dispositivo solo es compatible con L3, no podrá transmitir contenido HD protegido por DRM.

    Para obtener más detalles, consulte lo siguiente:

    • Documento de la solución DRM de Widevine
    • Documento de gestión de derechos digitales (DRM) de Android
    • Determinar el nivel de seguridad del dispositivo Android

    Reproducción sin conexión con DRM

    La reproducción sin conexión permite a los usuarios descargar contenido mientras están conectados y luego verlo cuando están desconectados. Para proteger su contenido, los SDK nativos utilizan lo siguiente:

    • El SDK nativo para Android usa Google Widevine
    • El SDK nativo para iOS usa FairPlay Streaming

    Para obtener más detalles, consulte la Descripción general de la reproducción sin conexión con los SDK del reproductor nativo.

    Otros dispositivos

    MPEG-DASH es compatible con muchos dispositivos, incluidos Android TV y Chromecast. Los videos empaquetados con DRM para el navegador de escritorio también se pueden reproducir en estos dispositivos.

    Muchos dispositivos de electrónica de consumo admiten una forma anterior de DRM: PlayReady sobre Smooth Streaming

    Registrarse en DRM

    DRM requiere una tarifa adicional. Comuníquese con su administrador de cuentas sobre la combinación de protección DRM para que pueda proteger su contenido donde sea que se reproduzca.

    Apple FairPlay

    FairPlay Streaming es un sistema DRM desarrollado por Apple para contenido de video en formato HLS.

    Para habilitar su cuenta para FairPlay, deberá:

    1. Obtenga una licencia FairPlay de Apple
    2. Póngase en contacto con el servicio de asistencia para configurar su cuenta de Video Cloud

    Problemas conocidos

    • Para la reproducción de Fairplay en dispositivos móviles y se utiliza una pantalla externa, el audio se reproducirá pero el video es solo una imagen en negro.

      Para más información, ver Seguridad de contenido (DRM) con los SDK del reproductor nativo

    Google Widevine Modular

    Widevine Modular es un sistema desarrollado por Google para proteger contenido de video en formato MPEG DASH.

    De forma predeterminada, Video Cloud utiliza el servidor de licencias Widevine de Brightcove.

    Microsoft PlayReady

    PlayReady es un sistema DRM desarrollado por Microsoft. Playready viene en dos formas:

    • Listo para jugar sin problemas - Proporciona protección DRM para contenido de transmisión fluida
    • PlayReady - Proporciona protección DRM para contenido MPEG-DASH

    Video Cloud utiliza el servidor de licencias PlayReady de Brightcove.

    Tenga en cuenta que existen límites de simultaneidad para PlayReady definidos en su contrato de servicio con Microsoft. Ver este documento de Microsoft para obtener información sobre la gestión de la concurrencia. Si se supera el límite de concurrencia de un vídeo, no se reproducirá para los espectadores más allá del número máximo.

    Configure su cuenta de Video Cloud

    El soporte de Brightcove y su administrador de cuentas deben configurar DRM para su cuenta. Comuníquese con su administrador de cuentas para obtener más detalles.
