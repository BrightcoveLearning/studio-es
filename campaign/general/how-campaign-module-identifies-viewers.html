El módulo Campaña intenta identificar a los espectadores mediante una variedad de fuentes de datos:

  • Cookie de campaña
  • GUID de Oracle Eloqua
  • Galleta Marketo Munchkin
  • Parámetro de URL de ID de Salesforce
  • Atributo HTML de ID de Salesforce
  • Cookie de HubSpot UTK
  • Parámetro de correo electrónico

El módulo Campaña se regirá por la configuración del navegador del usuario con respecto a No rastrear.

Nota: Al utilizar una integración personalizada, usted es responsable de identificar al espectador. Para obtener información, consulte Creación de integraciones personalizadas con el módulo Campaign.

Cookie de campaña

Jugadores habilitados para campaña establecerá una cookie de terceros y la utilizará para asociar los datos de visualización con el mismo usuario si el seguimiento de campañas está habilitado para su cuenta y el jugador lo está utilizando. Si un usuario alguna vez llena un formulario de cliente potencial de Campaign, Campaign creará un perfil para ese espectador y todos sus datos de visualización se asociarán con ese perfil.

La cookie que establecerá Campaign se llama _bc_uuid.

GUID de Oracle Eloqua

Los jugadores habilitados para la campaña con una integración de Eloqua invocarán las API de Eloqua para obtener la identificación del visitante, frecuentemente conocida como la GUID de Eloqua. Esta ID se captura en los datos de visualización del módulo Campaign y se utiliza para sincronizar los datos de visualización con Eloqua.

Eloqua utiliza su propio conjunto de cookies y cuando se configura un Brightcove Player con una integración de Eloqua, Campaign captura el GUID en la cookie de Eloqua y se comunica con Eloqua en el servidor para intentar averiguar quién es el usuario.

Galleta Marketo Munchkin

Los jugadores habilitados para campañas con una integración de Marketo se referirán a la cookie de Munchkin (_mkto_trk) que se crea en el navegador del usuario. Este valor se captura en los datos de visualización del módulo Campaign y se utiliza para sincronizar los datos de visualización con Marketo.

Parámetro de URL de ID de Salesforce

Para las integraciones de Salesforce, el complemento puede intentar identificar al espectador verificando la URL del parámetro llamado bcsfid. Esto permite a los usuarios de Salesforce enviar campañas de correo electrónico con una URL de página de destino que se genera de forma única con fines de seguimiento al proporcionar el ID de Salesforce del cliente potencial en la URL.

Atributo HTML de ID de Salesforce

Para las integraciones de Salesforce, si el complemento no puede identificar al espectador al verificar el bcsfid Parámetro de URL, buscará el data-bc-sfid atributo en el <video> elemento del código de publicación incrustado HTML para el video y use su valor como el ID de Salesforce del cliente potencial si está configurado.

Cookie de HubSpot UTK

Para las integraciones de HubSpot, el complemento inyecta el código de seguimiento JavaScript de HubSpot que suelta un hubspotutk cookie en el navegador del usuario. Esta cookie se utiliza para realizar un seguimiento de la identidad de un visitante dentro de HubSpot y asociar los datos de visualización de Brightcove con un contacto. Si el usuario ya tiene un hubspotutk cookie cuando se carga el reproductor, Campaign hará una solicitud de API a HubSpot y preguntará si el valor de la cookie corresponde a un contacto conocido, y si es así, Campaign asociará los datos de visualización con ese usuario.

Parámetro de correo electrónico

Al módulo Campaña también se le puede decir quién es el usuario a través de una dirección de correo electrónico. La campaña buscará un parámetro de URL titulado bcemail. Esto es útil con campañas de correo electrónico con una URL de página de destino que se ha generado de forma única con fines de seguimiento. Si vas a enviar un correo electrónico por lotes a una lista, ya sea desde una plataforma de marketing o desde un servicio de correo electrónico de terceros como MailChimp, deberás configurar el parámetro de correo electrónico en la configuración de la plataforma de envío para actualizar dinámicamente el enlace en el que hacen clic los usuarios. Luego, Campaign usará este parámetro de correo electrónico para buscar el lado del servidor del usuario y sincronizar los datos de visualización. El parámetro de correo electrónico también se puede incrustar con el código de incrustación del reproductor.

Ejemplo de URL

http://go.brightcove.com/be-video-hero-cust?bcemail=msmith@brightcove.com

Si el complemento Campaign no encuentra una dirección de correo electrónico a través del bcemail Parámetro de URL, luego verificará el <video> elemento del código de publicación HTML incrustado. Buscará el data-bc-email atributo en el elemento de video y use su valor como la dirección de correo electrónico si está configurado.

Ejemplo de incrustación

<video id = "player" data-account = "0000000001" data-player = "predeterminado" data-video-id = "0000000002"  data-bc-email= "msmith@brightcove.com" class = controles "video-js">< /video>

No rastrear

Los jugadores habilitados para la campaña cumplen por la configuración del navegador de un espectador con respecto a No rastrear (DNT) y no usará ni establecerá su cookie normal si ve que No rastrear está habilitado. También se puede indicar al complemento del reproductor que nunca realice un seguimiento si un cliente de Video Cloud desea desactivarlo para todos sus espectadores o usuario por usuario. Esto se hace usando el código de inserción en la página y configurando el data-do-not-track a un valor de 1.

<video-js data-video-id = "9240259371001" cuenta-de-datos = "3491906377" data-do-not-track = "1" data-player = "7dccc39d-0e2d-4b27-bea9-af44b2166148" data-embed = "default" class = "video-js" controles>< / video-js>< script src = "https://players.brightcove.net/1486906377/7dccc39d-0e2d-4b27-bea9-af44b2166148_default/index.min.js">< /texto>

Oracle Eloqua

Eloqua ha declarado que no procesan (básicamente ignoran), no rastrean las solicitudes y devolverán un GUID de Eloqua independientemente. Actualmente, Campaign no está manejando esto de ninguna manera especial, es decir, no usa la API de JavaScript de Eloqua cuando Campaign ve DNT. Eloqua también tiene soporte para un modo estricto que satisface los requisitos de privacidad de la UE. Cuando esto está habilitado para una cuenta de Eloqua, los usuarios finales no son rastreados a menos que se inscriban a través de un formulario.

Marketo

Marketo admite DNT, pero deja que el cliente lo active o desactive para su cuenta particular. Para obtener más información, consulte el documento de Marketo Editar la configuración de compatibilidad del navegador "No rastrear".

HubSpot

HubSpot respeta DNT según este documento.
