El módulo Campaign permite a Video Cloud sincronizar los datos de visualización de Brightcove Players con Oracle Eloqua. Una vez en Oracle Eloqua, estos datos se pueden utilizar para la segmentación, la lógica de la campaña, los informes, la puntuación de clientes potenciales y la personalización de las comunicaciones. La integración del módulo Campaign con Oracle Eloqua consta de dos componentes principales:

  • Un complemento de Brightcove Player para permitir el seguimiento de los datos de vistas de video
  • Un proceso de sincronización para enviar datos de seguimiento desde Video Cloud a Oracle Eloqua

Requisitos

Para utilizar el módulo Campaign con Oracle Eloqua, debe:

  • Tener una cuenta de Video Cloud profesional o empresarial
  • Habilite el módulo Campaña para su cuenta enviando la información requerida en la pantalla de presentación inicial que aparece cuando abre el módulo Campaña
  • Haga clic en el contrato de servicio la primera vez que utilice el módulo.
  • Utilice el código de publicación Advanced Embed (in-page) cuando publique su reproductor; El código de publicación de incrustación estándar (iframe) no funcionará ya que el complemento se ejecutará dentro de un iframe y, por lo tanto, no podrá acceder a ningún parámetro de URL de la página principal y es posible que no obtenga las cookies correctas.
  • Tenga Oracle Eloqua 10 y credenciales para iniciar sesión en: https://login.eloqua.com
    Nota: Estas credenciales se introducen en una página de inicio de sesión de Oracle Eloqua y no se proporcionan directamente a Brightcove.
  • La credencial que se utiliza para iniciar sesión en Oracle Eloqua debe agregarse al Usuarios avanzados: marketing group y tener los siguientes permisos:
    • Consumir API
    • Instalar aplicaciones en la nube
    • Configurar campañas de marketing

Características

Algunas de las características clave de la integración de Video Cloud - Oracle Eloqua son:

  • Formularios de clientes potenciales - Cree formularios Eloqua que aparecerán automáticamente cuando los espectadores vean videos. Para obtener información, consulte Creación de formularios de clientes potenciales personalizados para Oracle Eloqua.
  • Formato de datos - Los datos de visualización de la nube de video se envían a un objeto de datos personalizado (CDO) en Oracle Eloqua. Los datos que se replican en Oracle Eloqua son los siguientes:
    • Nombre del video
    • ID de video
    • ID de cuenta (Video Cloud)
    • URL de la página (URL de la página de referencia)
    • ID de jugador (reproductor de Video Cloud)
    • % Watched
  • Transferencia de datos - La visualización de datos se sincroniza cada hora y puede tardar de 2 a 4 horas en aparecer en Oracle Eloqua.
  • Retención de datos - Opcionalmente, los datos recopilados se pueden conservar en Video Cloud hasta por 1 mes.

Configurar la conexión

Antes de configurar la integración con Eloqua, le recomendamos que abra una nueva ventana del navegador y cierre la sesión manualmente de su cuenta Eloqua. Esto garantizará que cuando configure la integración, se le pedirá que inicie sesión en su cuenta de Eloqua y pueda confirmar que está utilizando la cuenta correcta.

Siga estos pasos para configurar la conexión entre Video Cloud y Oracle Eloqua.

  1. Abra el módulo Campaña y haga clic en Conexiones de datos en la navegación de la izquierda.
  2. Hacer clic Agregar nueva conexión.
  3. Seleccione Eloqua como la integración.
    Nota: Las conexiones de datos disponibles se basan en la suscripción a Video Cloud.
  4. (Opcional) Ingrese un Nombre de la conexión.
  5. Hacer clic Conectar.
  6. Hacer clic Conéctese a Eloqua.
  7. Se le pedirá que inicie sesión en Eloqua. Introduzca el nombre de su empresa, nombre de usuario y contraseña de Eloqua. Hacer clic Iniciar sesión.
  8. Si el inicio de sesión es exitoso, se le pedirá que instale el Brightcove Video Connect para Eloqua aplicación. Hacer clic Aceptar e instalar.
  9. Se le pedirá que habilite Eloqua para la cuenta. Hacer clic Permitir.
  10. Se le pedirá que permita Brightcove Video Connect para Eloqua para acceder a su cuenta Eloqua. Hacer clic Iniciar sesión.
  11. Confirme el nombre y el nombre de usuario de su empresa y haga clic en Aceptar.
  12. Se le pedirá que configure Eloqua. Hacer clic Configurar Eloqua.

Configurar la integración

Una vez realizada la conexión de Video Cloud a Oracle Eloqua, puede configurar la integración.

  1. Cheque Sincronizar vistas de video con Eloqua como actividades externas para sincronizar vistas como actividades externas. Las actividades externas son otro lugar para poner información en Eloqua. Los editores pueden optar por realizar informes o activar cosas en Eloqua mediante actividades externas. Para obtener más información, consulte Eloqua documentación.
  2. Por defecto, el Estado de conexión estarán Activado. La Estado de conexión determina si los datos de participación de video de Video Cloud se envían a Oracle Eloqua.
    • Activado - Los datos se recopilan y se enviarán a Eloqua
    • Discapacitado - No se recopilan datos (la integración está deshabilitada)
  3. Por defecto, Seguimiento de datos para espectadores desconocidos estarán Discapacitado. Esta configuración determina si los datos de participación de video se guardan en Video Cloud cuando se desconoce el espectador. Solo se envían a Eloqua datos de espectadores conocidos.
    • Activado - Los datos de todos los espectadores se recopilan y guardan en Video Cloud
    • Discapacitado - Los datos de espectadores desconocidos no se guardan en Video Cloud
  4. Seleccione una política de retención de datos. La política de retención de datos determina cuánto tiempo se conservarán los datos de visualización y los datos del formulario de cliente potencial (si utiliza un formulario de cliente potencial de la campaña) en su cuenta de Video Cloud. Retener datos en Video Cloud le permite recuperarse de sincronizaciones interrumpidas. Además, Campaign continuará reintentando eventos que no se sincronizan (generalmente porque el espectador no es un contacto conocido en el momento en que vio un video) hasta que el evento supere el período de retención de datos. Debe ajustar la configuración de retención de acuerdo con su política de privacidad. - no es necesario conservar ningún dato de usuario en Video Cloud. Los datos se eliminarán después del período de retención de datos.

Para eliminar la integración con Oracle Eloqua, haga clic en Desconectar. La parte inferior de la página enumerará todos los jugadores habilitados para la campaña y los formularios de clientes potenciales asociados.

El módulo Campaña admite múltiples conexiones a plataformas de marketing. Hacer clic Agregar nueva conexión para configurar integraciones adicionales.

Verificando la integración

Cuando la integración esté configurada, pruebe la integración para confirmar que los datos de visualización de video se envían a Eloqua. Para obtener información sobre cómo verificar la integración, consulte el Guía de pruebas de integración de Eloqua.

Creación de un reproductor habilitado para campaña

Los datos de participación de video solo se enviarán a Eloqua cuando esté utilizando un reproductor de video habilitado para Campaña. Para obtener información sobre cómo crear un Brightcove Player habilitado para la campaña, consulte Creación de un jugador habilitado para campaña.

Pasando una identificación de seguimiento

El módulo Campaña admite la capacidad de pasar un valor de ID de seguimiento a Eloqua. El ID de seguimiento se puede utilizar para realizar un seguimiento de un ID de campaña, por ejemplo. El ID de seguimiento se puede agregar a la URL que muestra el reproductor habilitado para la campaña o agregarse al código de inserción del reproductor.

  • Ejemplo: Anexado a la URL
  http://www.myurl.com/video.html?bctrackingid = 12345
  • Ejemplo: Agregado al código de inserción del jugador
  <video data-video-id = "6033402539001" data-account = "1486906377" data-player = "default" data-embed = "default" data-application-id class = "video-js" data-bc-trackingid = "12345" control S>< /video>< script src = "// players.brightcove.net/1486906377/default_default/index.min.js">< /texto>

La campaña luego pasará el valor a Eloqua como otro campo que está sincronizado.

Identificación de espectadores

Los jugadores habilitados para la campaña con una integración de Eloqua invocarán las API de Eloqua para obtener la identificación del visitante, frecuentemente conocida como la GUID de Eloqua. Esta ID se captura en los datos de visualización del módulo Campaign y se utiliza para sincronizar los datos de visualización con Eloqua.

Eloqua utiliza su propio conjunto de cookies y cuando se configura un Brightcove Player con una integración de Eloqua, Campaign captura el GUID en la cookie de Eloqua y se comunica con Eloqua en el servidor para intentar averiguar quién es el usuario.

La dirección de correo electrónico del usuario también se puede pasar y utilizar para identificar al espectador. Para obtener más información y un ejemplo, consulte Cómo el módulo de campaña identifica a los espectadores.
