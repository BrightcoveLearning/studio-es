El módulo Campaña permite a Video Cloud sincronizar los datos de visualización de la campaña habilitada Brightcove Players para plataformas de automatización de marketing populares. El módulo CampaignCampaign también se puede configurar con una integración personalizada. Con una integración personalizada, los datos de visualización se guardan localmente en Video Cloud y la API de audiencia se utiliza para extraer los datos mediante programación.

La integración del módulo Campaign consta de dos componentes principales:

  • Un complemento de Brightcove Player para permitir el seguimiento de los datos de vistas de video
  • Desarrollo personalizado que utiliza la API de audiencia para recuperar los datos de visualización.

Requisitos

Para utilizar el módulo Campaña debe:

  • Tener una cuenta de Video Cloud profesional o empresarial
  • Habilite el módulo Campaña para su cuenta enviando la información requerida en la pantalla de presentación inicial que aparece cuando abre el módulo Campaña
  • Haga clic en el contrato de servicio la primera vez que utilice el módulo.
  • Para poder recopilar clientes potenciales, el formulario de cliente potencial debe estar configurado y asignado a un jugador
  • Utilice el código de inserción Avanzado (en la página) al publicar su reproductor; El código de inserción estándar (iframe) no funcionará ya que el complemento se ejecutará dentro de un iframe y, por lo tanto, no podrá acceder a los parámetros de URL de la página principal

Configurar la conexión

Siga estos pasos para configurar una integración personalizada.

  1. Abra el módulo Campaña y haga clic en Conexiones de datos en la navegación de la izquierda.
  2. Hacer clic Agregar nueva conexión.
  3. Seleccione Personalizado como la integración.
    Nota: Las integraciones disponibles se basan en su suscripción a Video Cloud.
  4. (Opcional) Ingrese un Nombre de la conexión.
  5. Hacer clic Conectar.

La conexión personalizada ahora está configurada y lista para usar.

Configurar la integración

Una vez realizada la conexión, puede configurar la integración.

  1. Por defecto, el Estado de conexión estarán Activado. La Estado de conexión determina si se guardan los datos de participación de video de Video Cloud.
    • Activado - Los datos se recopilan y se guardarán
    • Discapacitado - No se recopilan datos (la integración está deshabilitada)
  2. Seleccione una política de retención de datos. La política de retención de datos determina cuánto tiempo se conservarán los datos de visualización y los datos del formulario de cliente potencial (si utiliza un formulario de cliente potencial de la campaña) en su cuenta de Video Cloud.
    Nota: Si selecciona Sin retención, NO se guardarán datos. Cuando uses la API para descargar datos, no habrá ninguno.

Creación de un reproductor habilitado para campaña

Los datos de participación en el video solo se guardarán cuando está utilizando un reproductor de video habilitado para Campaña. Para obtener información sobre cómo crear un Brightcove Player habilitado para la campaña, consulte Creación de un jugador habilitado para campaña.

Pasando una identificación de seguimiento

El módulo Campaña admite la capacidad de pasar un ID de seguimiento. El ID de seguimiento se puede utilizar para realizar un seguimiento de un ID de campaña, por ejemplo. El ID de seguimiento se puede agregar a la URL que muestra el reproductor habilitado para la campaña o agregarse al código de inserción del reproductor.

  • Ejemplo: Anexado a la URL
  http://www.myurl.com/video.html?bctrackingid = 12345
  • Ejemplo: Agregado al código de inserción del jugador
<video data-video-id = "6033402539001" data-account = "1486906377" data-player = "default" data-embed = "default" data-application-id class = "video-js" data-bc-trackingid = "12345" control S>< /video>< script src = "// players.brightcove.net/1486906377/default_default/index.min.js">< /texto>

Identificación de espectadores

Al utilizar una integración personalizada, usted es responsable de identificar a los espectadores. El complemento Campaign necesita dos datos clave:

Descripción Parámetro de URL Insertar parámetro Valor
¿Se conoce al espectador (necesario para el formulario de cliente potencial)? bcknownuser data-bc-known-user 0 o 1
¿Cuál es el GUID (necesario para ver el seguimiento de eventos)? bccustomguid data-bc-custom-guid string
 

Estos valores se pueden pasar a través de parámetros de URL o mediante el código de inserción del reproductor. Para que se produzca el seguimiento del espectador, la identificación de usuario debe pasarse como un parámetro de URL en la página que muestra el video. El parámetro de usuario conocido es necesario para los formularios de clientes potenciales (si desea que el formulario de clientes potenciales se muestre condicionalmente para espectadores desconocidos).

Nota: Para que el reproductor / complemento lea los parámetros de URL, se debe utilizar el código de inserción Avanzado (en la página). El código de incrustación estándar (iframe) no funcionará ya que el complemento se ejecutará dentro de un iframe y, por lo tanto, no puede acceder a la URL de la página principal que tiene el parámetro.

Ver datos de video

Hay varias opciones disponibles para ver datos de video cuando se usa una integración personalizada:

  • Actividades del espectador (eventos) - Proporcionada como parte del módulo Campaña, esta opción se puede utilizar para ver datos de video dentro del módulo Campaña. Para obtener información, consulte Visualización de eventos de jugadores en el módulo de campaña.
  • Exportar datos - Proporcionada como parte del módulo Campaña, esta opción se puede utilizar para exportar la vista de video y los datos de clientes potenciales. Para obtener información, consulte Exportación de datos de campaña desde el módulo de campaña.
  • API de audiencia - La API Audience se puede utilizar para recuperar de manera programática la vista de video y los datos de clientes potenciales. Para obtener información, consulte Descripción general: API de audiencia.
